## 📦 INCLUDED FEATURES

### ✅ Producer Service (REST API)
- **Endpoint:** `POST /api/orders`
- **Auto-generated Order IDs:** UUID-based (ORD-XXXXXXXX)
- **Input Validation:** User and item fields required
- **Response Status:** 202 Accepted
- **Error Handling:** 400 Bad Request for validation errors

### ✅ Kafka Configuration
- **Topic:** "orders"
- **Partitions:** 3 (for parallel processing)
- **Replication Factor:** 1 (for local development)
- **Bootstrap Servers:** localhost:9092
- **Serialization:** JSON for both producer and consumer
- **Auto Topic Creation:** Enabled

### ✅ Consumer Service
- **Listener:** @KafkaListener on "orders" topic
- **Consumer Group:** order-consumer-group
- **Concurrency:** 3 concurrent consumers
- **Auto-commit:** Enabled for offset tracking
- **Processing:** Logs complete order flow with emojis
- **Error Handling:** Exception logging with details

### ✅ Data Model
- `Order` class with: orderId, user, item, timestamp, status
- `ApiResponse` generic wrapper for consistent API responses
- Both use Lombok for boilerplate reduction
- Full Jackson JSON support

### ✅ Configuration Management
- **Default (application.yml):** Base configuration
- **Development (application-dev.yml):** DEBUG logging
- **Production (application-prod.yml):** Optimized settings
- **Spring Profiles:** Active via -Dspring.profiles.active=dev/prod

### ✅ Docker Support
- **docker-compose.yml:** Kafka + Zookeeper one-command startup
- **Dockerfile:** Multi-stage build for optimized image
- **Health Checks:** Container readiness probes
- **Networking:** Bridged network for container communication

## 📋 EXAMPLE API CALLS

### 1. Health Check
```bash
curl http://localhost:8080/api/orders/health
```
Response: `{"status": "✅ Order Service is running", ...}`

### 2. Create Order (Auto-generated ID)
```bash
curl -X POST http://localhost:8080/api/orders \
  -H "Content-Type: application/json" \
  -d '{"user": "alice@example.com", "item": "Phone"}'
```
Response: `{"status": "success", "orderId": "ORD-ABC12345", ...}`

### 3. Create Order (Specified ID)
```bash
curl -X POST http://localhost:8080/api/orders \
  -H "Content-Type: application/json" \
  -d '{"orderId": "ORD-001", "user": "bob@example.com", "item": "Tablet"}'
```

### 4. Get Order Status
```bash
curl http://localhost:8080/api/orders/ORD-001
```
Response: `{"orderId": "ORD-001", "status": "PROCESSING", ...}`

### Manual Kafka Verification
```bash
# List topics
docker exec kafka-broker kafka-topics --list --bootstrap-server localhost:9092

# View topic details
docker exec kafka-broker kafka-topics --describe --topic orders --bootstrap-server localhost:9092

# Read messages from topic
docker exec kafka-broker kafka-console-consumer \
  --bootstrap-server localhost:9092 \
  --topic orders \
  --from-beginning \
  --max-messages 5

# Check consumer group
docker exec kafka-broker kafka-consumer-groups --describe \
  --group order-consumer-group \
  --bootstrap-server localhost:9092
```
